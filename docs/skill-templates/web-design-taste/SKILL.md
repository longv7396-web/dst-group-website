# Web Design Taste

Use this skill to raise the perceived quality of a web interface. The goal is for the site to feel like a real company product, not a student exercise or AI template.

## Taste Standards

- Make the page feel intentional, calm, and credible.
- Prefer fewer stronger elements over many weak decorative elements.
- Use contrast to guide attention, not to make every element loud.
- Treat whitespace as structure. Do not cram sections together.
- Ensure every card, button, image, tag, and divider has a clear reason to exist.
- Use hover states that feel expensive: slight lift, subtle border, controlled glow, no jumpy layout shift.
- Use motion as pacing. Do not add animation that distracts from content.

## Audit Pass

Review these before making changes:

- Spacing: Are section paddings, card gaps, and text margins consistent?
- Hierarchy: Is the primary message obvious within three seconds?
- Typography: Are headings strong, body copy readable, and labels not oversized?
- Contrast: Can text be read comfortably on dark backgrounds and images?
- Imagery: Are images clear, direct, and treated consistently?
- Cards: Are card heights, borders, radius, shadows, and content positions balanced?
- CTA: Is the main action visible and not buried under visual noise?
- Animation: Does motion improve rhythm without causing lag or layout shift?
- Responsive: Does the design hold up at 390px, 768px, 1366px, and 1920px?

## Premium Dark Theme Guidance

- Keep the base dark palette rich but not muddy.
- Use gold/orange accents sparingly for emphasis, CTA, and focus.
- Avoid covering the page with heavy glow, bokeh, diagonal lines, or noisy patterns.
- Use subtle borders and low-opacity surfaces to separate content.
- If using gradients, keep them soft and purposeful.

## Decision Rules

- If a visual effect makes real content harder to see, remove or reduce it.
- If a section looks generic, strengthen the concept before adding decoration.
- If an element does not improve comprehension, conversion, or brand feel, simplify it.
- If a design choice creates mobile risk, choose the more stable layout.

## Completion Criteria

- The interface feels coherent and premium.
- The user can scan the page and understand the offer.
- UI elements do not overlap, overflow, or fight for attention.
- The result remains maintainable in the existing React/CSS structure.
