# Frontend Development

Read the existing project before editing. Follow current patterns when they exist.

Prefer small, named components and data-driven rendering over repeated markup. Keep data and source metadata separate from presentation components.

For each implementation:
- Use semantic HTML and meaningful alt text.
- Keep interactions keyboard-accessible.
- Keep state minimal and local unless shared state is clearly needed.
- Avoid unused dependencies and unrelated refactors.
- Run build and lint checks when available, then fix surfaced issues.

Do not leave lorem ipsum, sample users, random external images, or unused generated files in the final app.
